## Описание

Этот проект представляет собой backend-часть для игры "Три в ряд". Приложение реализует логику игры с несколькими уровнями сложности, таблицу лидеров, обработку свапов и обновлений поля, а также автоматическая маркировка проигрыша для неактивных игр.

Игра поддерживает конфигурацию уровней (простой, средний, сложный, кастомный), где предметы обозначаются буквами алфавита (с расширением как в Excel). Включает проверку на возможность хода, цепные реакции уничтожений, и Swagger-документацию для API.

## Демонстрация
API доступно для тестирования через документацию: [Swagger UI](http://77.239.98.1/docs)

## Функции

- **Уровни сложности**:
  - Простой: поле 12x12, цель — 40 очков.
  - Средний: поле 8x8, цель — 50 очков, ограничение одним свапом.
  - Сложный: поле 8x8, цель — 25 очков, одним свапом, случайный предмет.
  - Кастомный: пользовательский размер поля, настройки (случайный предмет, один свап, количество типов предметов, целевое кол-во очков).

- **Логика игры**:
  - Создание игры с конфигурацией.
  - Свап элементов (с направлениями: вверх, вниз, влево, вправо) с цепными реакциями (полными обновлениями поля без потери очков, если нет возмождных ходов) и снимками поля после каждого падения.
  - Обновление поля (refresh) с потерей очков.
  - Получение состояния игры, очков, лидерборда (топ-10 по времени).

- **Дополнительно**:
  - Таймаут: игра проигрывается, если нет действий в течение 2-х часов (использует APScheduler).
  - Таблица лидеров: топ-10 результатов по сложности, отсортированные по времени игры.

## Технологический стек

## Установка

1. **Клонируйте репозиторий**:
   ```
   git clone https://github.com/StupStuP21/ThreeInRowBackend.git
   cd ThreeInRowBackend
   ```

2. **Создайте виртуальное окружение**:
   ```
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   ```

3. **Установите зависимости**:
   ```
   pip install -r requirements.txt
   ```

4. **Настройте базу данных и конфигурацию**:
   - Установите PostgreSQL и создайте БД (по умолчанию `three-row-game`).
   - Файл конфигурации `config.toml` генерируется автоматически при первом запуске приложения с помощью `project_settings_parser.py` (с дефолтными значениями). Если файл уже существует, он не перезаписывается.
   - Вы можете отредактировать `config.toml` для кастомизации. Структура файла:
     ```toml
     [db_connection]
     db_name = "three-row-game"  # Название базы данных
     db_user = "postgres"        # Пользователь БД
     db_password = "user"        # Пароль пользователя БД
     db_host = "localhost"       # Хост БД
     db_port = 5432              # Порт БД

     [front_connection]
     hosts = [ "http://localhost:5173", "http://127.0.0.1:5173" ]  # Список хостов для фронтенда (CORS)
     ```
   - Если нужно перегенерировать файл с новыми дефолтами, используйте `create_settings_file(replace_if_exist=True)` в `SettingsParser`.

5. **Примените миграции**:
   ```
   alembic upgrade head
   ```

6. **Запустите приложение**:
   ```
   uvicorn main:app --reload
   ```
   Приложение доступно на `http://localhost:8000`. При первом запуске проверьте генерацию `config.toml` и подключение к БД.

## Использование

- **Документация API**: Перейдите на `http://localhost:8000/docs` для Swagger UI, где можно тестировать эндпоинты.

### Основные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/create_game/create` | Создание новой игры |
| GET | `/difficulties/` | Получение списка уровней сложности |
| GET | `/difficulties/{difficulty_id}` | Получение детали уровня сложности по ID |
| GET | `/game/{game_id}` | Получение текущего состояния игры |
| PATCH | `/game/{game_id}` | Обновление полей игры |
| GET | `/game/{game_id}/score` | Получение текущих очков игры |
| POST | `/game/{game_id}/swap` | Выполнение свапа |
| POST | `/game/{game_id}/refresh` | Полное обновление поля |
| GET | `/leaderboard/{difficulty_id}` | Получение лидерборда для уровня сложности |

Для детальной схемы запросов/ответов используйте Swagger.
